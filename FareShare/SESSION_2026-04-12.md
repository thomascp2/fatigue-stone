### Pipeline Flow (Phase 1)

```
Upload PDF/image
    → Claude Sonnet Vision extracts parcel fields
    → Alabama GIS ArcGIS REST fetches comparable parcels
    → Comparison logic: flag if assessed value >15% above median comp
    → Dispute Report generated (Markdown + PDF)
    → Dispute record written to SQLite (→ PostgreSQL in prod)
```

### State Machine

```
uploaded → analyzing → flagged / not_flagged
    → letter_drafted → letter_sent → pending_decision
    → resolved_win / resolved_loss
```

### Key Design Decisions

- **Claude only** — no model split. Sonnet handles both vision extraction and downstream reasoning in one call. Eliminates latency and failure points.
- **SQLite for now** — one-line swap to PostgreSQL via SQLAlchemy. No reason to set up Postgres before Phase 0 validation.
- **Alabama GIS first** — free, no API key, publicly accessible ArcGIS REST endpoints for Madison/Limestone/Morgan counties. ATTOM is a paid fallback for when free data is insufficient.
- **15% threshold** — aligns with Alabama BOE case law on meaningful over-assessment. Configurable constant in `comps.py`.
- **CLI analyze.py** — lets you run Phase 0 manual validation against real assessment notices before the web server is needed.

## How to Run

### Phase 0 Manual Validation (CLI)
```bash
cd backend
pip install -r requirements.txt
cp ../.env.example ../.env   # add ANTHROPIC_API_KEY
python analyze.py path/to/assessment.pdf
```

### API Server
```bash
cd backend
uvicorn main:app --reload
# Docs at http://localhost:8000/docs
```
